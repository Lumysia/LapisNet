## Reporting a Vulnerability

We value the security of our project and the community. If you discover a vulnerability, please **report it to us immediately** so we can address it as quickly as possible.

### How to Report

Please report security vulnerabilities via **private disclosure** by sending an email to our dedicated security address: `security@lapisnet.app`.

**Do not** open a public GitHub issue for security-related matters.

### Reporting Guidelines

To help us triage and resolve your report efficiently, please include the following details:

- A clear **description** of the vulnerability.
- **Steps to reproduce** the issue (including any necessary code or configuration).
- The **affected version(s)** of the project.
- The **potential impact** of the vulnerability.
- Any suggested **mitigation or fix**, if known.

### Our Response

1.  **Acknowledgment:** We aim to acknowledge receipt of your report within **48 hours**.
2.  **Assessment:** Our team will investigate and confirm the vulnerability. We will provide an update on the assessment status within **7 days**.
3.  **Resolution:** For accepted vulnerabilities, we will prioritize a fix and aim to release a patch or new version as soon as possible, striving for a resolution within **30 days**. We will coordinate the disclosure timeline with you.
4.  **Public Disclosure:** Once a fix is available, we will disclose the vulnerability publicly via a **GitHub Security Advisory**. We credit reporters who wish to be named.

We greatly appreciate the efforts of security researchers in making our project safer.
